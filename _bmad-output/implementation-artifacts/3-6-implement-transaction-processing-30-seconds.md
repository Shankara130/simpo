## Story

**As a** Cashier,
**I want** to complete end-to-end sales transactions within 30 seconds from product scan to receipt print,
**So that** I can serve customers efficiently even during peak hours.

---

## Acceptance Criteria

1. **AC1: Backend Transaction Endpoint (<15 seconds for scan + cart)**
   - Backend API endpoint `POST /api/v1/transactions` accepts transaction data
   - Endpoint validates all products exist and have sufficient stock before processing
   - Endpoint performs atomic operations: deduct stock, create transaction, create items
   - Transaction uses database transaction with rollback on any error
   - Endpoint returns created transaction with transaction number and ID
   - Response time <2 seconds for typical 10-item transaction

2. **AC2: Mobile Transaction Service Integration**
   - Mobile TransactionService calls backend API with cart and payment data
   - Service converts CartContext items to SaleRequest format
   - Service converts PaymentData to backend payment method format
   - Service handles API errors with Indonesian error messages
   - Service includes JWT authentication from AsyncStorage
   - Service timeout set to 15 seconds (allows for network latency)

3. **AC3: POSScreen Transaction Flow Integration**
   - Transaction creation triggered after payment confirmation in POSScreen
   - Cart data converted to backend format (items with productId, quantity, unitPrice)
   - Payment data converted to backend format (CASH, TRANSFER, E-WALLET)
   - Stock validation on backend prevents overselling (double-check with mobile)
   - Transaction number generated by backend (format: TRX-YYYYMMDD-XXXX)
   - Cart cleared ONLY after successful transaction creation

4. **AC4: Transaction Completion Time Tracking**
   - System tracks transaction duration from first scan to completion
   - Timer starts when first product is added to cart
   - Timer ends when backend confirms transaction creation
   - Transaction duration logged with transaction record
   - Performance data available for monitoring (NFR-PERF-001 compliance)

5. **AC5: Error Handling and Recovery**
   - Backend returns RFC 7807 error responses for validation failures
   - Mobile displays Indonesian error messages for common failures:
     - "Produk stok habis" (product out of stock)
     - "Stok tidak mencukupi" (insufficient stock)
     - "Koneksi gagal, coba lagi" (connection failed)
   - Transaction NOT created if any item has insufficient stock
   - Cart preserved on error (user can retry or modify cart)
   - Retry option available without re-entering payment data

6. **AC6: Performance Requirements (<30 seconds total)**
   - Product scanning and cart building: <15 seconds ( Stories 3.1-3.3 )
   - Payment processing: <10 seconds (Story 3.4)
   - Transaction API call: <2 seconds (this story)
   - Receipt printing: <5 seconds (Story 3.5)
   - **Total: <32 seconds** (within acceptable margin of 30-second target)

**Backend Transaction Handler:**

```go
// transaction_handler.go
package handlers

import (
    "github.com/gin-gonic/gin"
    "github.com/vahiiiid/go-rest-api-boilerplate/internal/services"
)

type TransactionHandler struct {
    transactionService services.TransactionService
}

func NewTransactionHandler(transactionService services.TransactionService) *TransactionHandler {
    return &TransactionHandler{transactionService: transactionService}
}

// CreateTransaction handles POST /api/v1/transactions
func (h *TransactionHandler) CreateTransaction(c *gin.Context) {
    // 1. Get cashier ID from JWT context
    cashierID, exists := c.Get("userID")
    if !exists {
        c.JSON(401, gin.H{"type": "https://api.simpo.com/errors/unauthorized", "title": "Unauthorized", "status": 401})
        return
    }

    // 2. Get branch ID from JWT context or request
    branchID, exists := c.Get("branchID")
    if !exists {
        c.JSON(400, gin.H{"type": "https://api.simpo.com/errors/missing-branch", "title": "Branch ID required", "status": 400})
        return
    }

    // 3. Parse SaleRequest from request body
    var saleRequest services.SaleRequest
    if err := c.ShouldBindJSON(&saleRequest); err != nil {
        c.JSON(400, gin.H{"type": "https://api.simpo.com/errors/invalid-request", "title": "Invalid request", "status": 400, "detail": err.Error()})
        return
    }

    // 4. Validate sale request
    if len(saleRequest.Items) == 0 {
        c.JSON(400, gin.H{"type": "https://api.simpo.com/errors/empty-cart", "title": "Cart cannot be empty", "status": 400})
        return
    }

    // 5. Call ProcessSale with transactional operations
    transaction, err := h.transactionService.ProcessSale(c.Request.Context(), &saleRequest, cashierID.(uint), branchID.(uint))
    if err != nil {
        // Return RFC 7807 error response
        c.JSON(400, gin.H{
            "type": "https://api.simpo.com/errors/transaction-failed",
            "title": "Transaction Failed",
            "status": 400,
            "detail": err.Error(),
            "instance": "/api/v1/transactions",
        })
        return
    }

    // 6. Return created transaction
    c.JSON(201, transaction)
}
```

**Mobile TransactionService:**

```typescript
// TransactionService.ts
import axios, { AxiosError } from 'axios';
import AsyncStorage from '@react-native-async-storage/async-storage';
import { TransactionRequest, TransactionResponse, TransactionError } from '../types/transaction.types';
import { CartItem } from '../types/cart.types';
import { PaymentData } from '../types/payment.types';

const API_BASE_URL = __DEV__ ? 'http://localhost:8080' : 'https://api.simpo.id';
const API_VERSION = '/api/v1';
const FULL_API_URL = `${API_BASE_URL}${API_VERSION}`;

export class TransactionServiceError extends Error {
  constructor(
    message: string,
    public statusCode?: number,
    public originalError?: any
  ) {
    super(message);
    this.name = 'TransactionServiceError';
  }
}

export const TransactionService = {
  /**
   * Create a new transaction
   */
  createTransaction: async (cartItems: CartItem[], paymentData: PaymentData): Promise<TransactionResponse> => {
    try {
      // 1. Get JWT token from AsyncStorage
      const token = await AsyncStorage.getItem('jwt_token');
      if (!token) {
        throw new TransactionServiceError('Not authenticated', 401);
      }

      // 2. Convert CartContext items to SaleRequest format
      const items = cartItems.map(item => ({
        productId: item.productId,
        quantity: item.quantity,
        unitPrice: item.price.toString(),
      }));

      // 3. Convert PaymentData to backend payment method
      const paymentMethod = mapPaymentMethod(paymentData);

      // 4. Build request body
      const requestBody: TransactionRequest = {
        items,
        paymentMethod,
        // Optional fields
        customerName: null,
        taxAmount: '0',
        discountAmount: '0',
      };

      // 5. Call backend API
      const response = await axios.post<TransactionResponse>(
        `${FULL_API_URL}/transactions`,
        requestBody,
        {
          headers: {
            'Authorization': `Bearer ${token}`,
            'Content-Type': 'application/json',
          },
          timeout: 15000, // 15 seconds
        }
      );

      return response.data;
    } catch (error) {
      if (axios.isAxiosError(error)) {
        const apiError = error as AxiosError<any>;
        if (apiError.response?.data) {
          // Handle RFC 7807 error response
          const errorData = apiError.response.data;
          throw new TransactionServiceError(
            mapErrorMessage(errorData.detail || errorData.title),
            apiError.response.status,
            errorData
          );
        }
      }
      throw TransactionService._handleError(error, 'Failed to create transaction');
    }
  },

  _handleError: (error: any, defaultMessage: string): TransactionServiceError => {
    if (error instanceof TransactionServiceError) {
      return error;
    }
    return new TransactionServiceError(defaultMessage, 500, error);
  },
};

/**
 * Map PaymentData to backend payment method format
 */
function mapPaymentMethod(paymentData: PaymentData): string {
  switch (paymentData.method) {
    case 'CASH':
      return 'CASH';
    case 'TRANSFER':
      return 'TRANSFER';
    case 'E_WALLET':
      return 'E-WALLET';
    default:
      throw new TransactionServiceError('Invalid payment method', 400);
  }
}

/**
 * Map backend error messages to Indonesian
 */
function mapErrorMessage(message: string): string {
  if (message.includes('insufficient stock') || message.includes('stok tidak mencukupi')) {
    return 'Stok tidak mencukupi untuk menyelesaikan transaksi ini';
  }
  if (message.includes('out of stock') || message.includes('stok habis')) {
    return 'Salah satu produk sudah habis stoknya';
  }
  if (message.includes('not found') || message.includes('tidak ditemukan')) {
    return 'Produk tidak ditemukan';
  }
  if (message.includes('unauthorized') || message.includes('tidak diotorisasi')) {
    return 'Sesi Anda telah berakhir. Silakan login kembali';
  }
  if (message.includes('network') || message.includes('koneksi')) {
    return 'Koneksi gagal. Periksa koneksi internet Anda dan coba lagi';
  }
  return 'Gagal membuat transaksi. Silakan coba lagi.';
}
```

### Implementation Gotchas

**⚠️ CRITICAL: Database Transaction with Rollback**

Backend ProcessSale MUST use database transactions with rollback:
```go
// CORRECT: Use database transaction with rollback
func (s *TransactionServiceImpl) ProcessSale(ctx context.Context, sale *SaleRequest, cashierID, branchID uint) (*Transaction, error) {
    return s.db.Transaction(func(tx *gorm.DB) (*Transaction, error) {
        // 1. Lock and validate stock for all items
        // 2. Deduct stock
        // 3. Create transaction
        // 4. Create transaction items
        // If any step fails, entire transaction rolls back
    })
}

// WRONG: No transaction wrapper
func (s *TransactionServiceImpl) ProcessSale(...) (*Transaction, error) {
    // Stock deduction might succeed but transaction creation might fail
    // This leads to stock inconsistency!
}
```

**⚠️ CRITICAL: Stock Level Locking**

Prevent race condition when multiple cashiers sell same product:
```go
// CORRECT: Use SELECT FOR UPDATE to lock rows
rows := tx.Raw("SELECT * FROM products WHERE id = ? FOR UPDATE", productID).Scan(&product)

// WRONG: Read without locking
rows := tx.Find(&product, productID)
// Another cashier might read same stock level before we deduct
```

**⚠️ CRITICAL: JWT Token in Mobile Request**

Mobile must include JWT token in Authorization header:
```typescript
// CORRECT: Include JWT token from AsyncStorage
const token = await AsyncStorage.getItem('jwt_token');
const response = await axios.post(url, data, {
  headers: { 'Authorization': `Bearer ${token}` }
});

// WRONG: No authentication
const response = await axios.post(url, data);
// Backend will return 401 Unauthorized
```

**⚠️ CRITICAL: Cart to Backend Mapping**

Cart items must be correctly mapped to backend format:
```typescript
// CORRECT: Map CartItem to TransactionItemRequest
const items = cartItems.map(item => ({
  productId: item.productId,    // Number from Product.id
  quantity: item.quantity,      // Number from Cart state
  unitPrice: item.price.toString(),  // String for precision
}));

// WRONG: Use wrong field names or types
const items = cartItems.map(item => ({
  product_id: item.productId,  // Wrong! Backend uses camelCase
  qty: item.quantity,          // Wrong! Backend uses 'quantity'
  price: item.price,           // Wrong! Backend expects string
}));
```

**⚠️ CRITICAL: Payment Method Mapping**

PaymentData discriminated union must map to backend format:
```typescript
// CORRECT: Map PaymentData to backend payment method
function mapPaymentMethod(paymentData: PaymentData): string {
  switch (paymentData.method) {
    case 'CASH': return 'CASH';
    case 'TRANSFER': return 'TRANSFER';
    case 'E_WALLET': return 'E-WALLET';  // Note the hyphen!
  }
}

// WRONG: Use frontend method directly
const paymentMethod = paymentData.method;  // 'E_WALLET' doesn't match backend 'E-WALLET'
```

**⚠️ CRITICAL: Transaction Number from Backend**

Use backend-generated transaction number, not client-generated:
```typescript
// CORRECT: Use backend transaction number
const transaction = await TransactionService.createTransaction(items, paymentData);
const receiptData = {
  transactionNumber: transaction.transactionNumber,  // From backend
  // ...
};

// WRONG: Generate transaction number on client
const transactionNumber = `TRX-${formatDate(new Date())}-${nextNumber}`;
const receiptData = { transactionNumber, ... };
// This breaks audit trail consistency!
```

**⚠️ CRITICAL: Cart Clear After Transaction Success Only**

Cart should only be cleared after successful transaction creation AND receipt printing:
```typescript
// CORRECT: Clear cart after both transaction and printing succeed
const handlePaymentConfirmed = async (paymentData: PaymentData) => {
  try {
    // 1. Create transaction
    const transaction = await TransactionService.createTransaction(items, paymentData);
    
    // 2. Print receipt
    await printReceipt(generateReceiptData(items, paymentData, transaction.transactionNumber));
    
    // 3. Only clear cart AFTER both succeed
    actions.clearCart();
    showSuccess('Transaction complete');
  } catch (error) {
    showError(error.message);
    // Cart is NOT cleared - user can retry
  }
};

// WRONG: Clear cart before transaction completes
const handlePaymentConfirmed = async (paymentData: PaymentData) => {
  actions.clearCart();  // Cart cleared too early!
  const transaction = await TransactionService.createTransaction([], paymentData);
  // No items to process!
};
```

**⚠️ CRITICAL: Error Message Localization**

Backend errors must be mapped to Indonesian for users:
```typescript
// CORRECT: Map backend errors to Indonesian messages
function mapErrorMessage(message: string): string {
```
